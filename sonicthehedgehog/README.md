# Sonic the Hedgehog — Recompiler Output

Generated output and configuration for running Sonic the Hedgehog (Genesis) through the [GenesisRecomp](../genesisrecomp/) static recompiler. This is a **tech demo with known bugs** — paired with the [SonicTheHedgehogRecomp](../SonicTheHedgehogRecomp/) runner, it boots to the title screen, loads Green Hill Zone, and gameplay runs, but jumping doesn't work and audio is garbled.

## What's In This Repo

| File/Directory | Purpose |
|----------------|---------|
| `generated/sonic_full.c` | 337 native C functions generated from the Sonic 1 ROM (4.3 MB, **READ-ONLY**) |
| `generated/sonic_dispatch.c` | Address-to-function dispatch table (40K, **READ-ONLY**) |
| `game.cfg` | Recompiler configuration: 337 extra_func entries, yield hints, VBlank yield points |
| `extras.c` / `extras.h` | Game-specific hooks: entry point routing, CRC32 check, sound driver init |
| `annotations_from_disasm.csv` | Function name annotations from the Sonic 1 disassembly |

## READ-ONLY: Do Not Edit Generated Files

`sonic_full.c` and `sonic_dispatch.c` are generated by the recompiler. **Never edit them directly.**

- Codegen bugs → fix in the [recompiler](../genesisrecomp/recompiler/src/code_generator.c)
- Missing functions → add `extra_func` entries to `game.cfg`, then regenerate
- Game-specific workarounds → fix in the [runner's glue.c](../SonicTheHedgehogRecomp/runner/glue.c)

## Regenerating

```bash
# Build the recompiler first (see genesisrecomp/README.md)
../genesisrecomp/build/recompiler/Release/GenesisRecomp.exe <sonic.bin> --game game.cfg
```

After regenerating, rebuild the game runner.
